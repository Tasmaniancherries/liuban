# Security policy

## Supported versions

We release the mobile app as discrete builds (not a separately versioned library).
Security fixes land on the `main` branch and ship in the next store release.

## Reporting a vulnerability

Please **do not** open a public GitHub issue for undisclosed security problems.

Instead, contact the maintainers privately (for example via email, if published on the
GitHub organization or maintainer profile). Include:

- A short description of the issue and affected component (client app, API surface, etc.)
- Steps to reproduce or a proof-of-concept, if safe to share
- Likely impact and any suggested mitigation you may have

We aim to acknowledge serious reports within a few business days and coordinate disclosure.

## 以中文回報（概要）
