# 留伴 App 上線候選版本說明（Launch Candidate）

版本狀態：候選上線版（待完成 `docs/release_readiness_checklist.md` 內 P0 驗收）

## 本版重點

- 全面移除頁面載入失敗時的本地示例資料顯示，統一改為「空狀態 + 明確錯誤提示 + 可重試」。
- 訊息、發帖、好友、認證等高頻輸入入口補齊長度邊界與提交前校驗，減少超長輸入導致的提交失敗與異常。
- 主要錯誤提示與無障礙語義文案已與實際行為對齊，避免「仍會顯示本地示例內容」的誤導描述。
- 測試用例已同步更新並擴充，覆蓋失敗路徑、空狀態與新增輸入邊界保護。

## 使用者可感知變更

- 載入失敗時，不再展示示例內容；會直接看到空狀態與提示訊息。
- 私訊/客服/發帖輸入框新增字數上限，空內容送出會被即時阻止。
- 註冊、登入、忘記密碼、重設密碼、修改密碼、加好友等表單新增長度邊界與明確提示。
- 多數詳情頁與列表頁在錯誤狀態下保留下拉重試能力。

## 品質與回歸

- 本輪改動已完成針對性 widget 測試與全量 `flutter test` 回歸。
- 相關變更檔案無新增 lint 錯誤（`ReadLints` 檢查通過）。

## 風險與關注點

- 目前主要風險已集中到發佈流程與環境因素（真機差異、弱網、後端聯調狀態），非前端資料展示邏輯。
- 建議上線前按清單完成：雙平台真機回歸、後端簽收、監控接入、發佈鏈路與隱私合規核對。

## 發佈建議

- 對外公告建議聚焦「穩定性提升與錯誤提示改進」，避免提及開發期示例資料機制。
- 建議採灰度發布並設定回滾閾值（崩潰率、登入成功率、關鍵路徑成功率）。
