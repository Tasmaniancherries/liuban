# 留伴 App 上线准备清单

本清单用于将当前可运行版本推进到可正式发布状态。建议按 P0 -> P1 -> P2 执行。

## P0（上线阻塞项）

- [ ] 双平台真机回归（iOS / Android）
  - 验收：注册、登录、发帖、消息、好友、推广、设置主流程可走通。
- [ ] 后端联调签收
  - 验收：关键接口成功/失败/空数据/超时场景均通过；错误提示符合预期。
- [ ] 崩溃与错误监控接入
  - 验收：可在监控平台看到崩溃、异常日志和版本号维度。
- [ ] 发布构建链路验证
  - 验收：可重复产出可安装包；签名、渠道、版本号策略可追溯。
- [ ] 隐私与合规入口检查
  - 验收：权限说明、隐私政策、用户协议在应用内可达且文案一致。

## P1（上线质量项）

- [ ] 弱网专项测试
  - 验收：断网、超时、抖动网络下页面能稳定显示空状态和可重试入口。
- [ ] 可观测性补齐
  - 验收：关键行为埋点（注册、登录、发帖、消息）可用于漏斗分析。
- [ ] 运营与客服应急流程演练
  - 验收：出现线上问题时，客服和研发可在 30 分钟内定位模块并给到用户反馈。
- [ ] 灰度发布策略确认
  - 验收：首批灰度比例、回滚条件、观测窗口和负责人已明确。

## P2（发布后 1-2 周）

- [ ] 指标复盘
  - 验收：D1 留存、注册转化、发帖转化、消息活跃有首轮数据结论。
- [ ] 性能优化排期
  - 验收：首屏耗时、列表滚动、图片加载三项有量化优化目标。
- [ ] 问题闭环机制
  - 验收：高频问题具备工单分类、修复 SLA、版本归档记录。

## 发布前最终核对（建议 T-1）

- [ ] `dart analyze` 通过
- [ ] `flutter test` 全量通过
- [ ] 版本说明（对内/对外）已准备
- [ ] 回滚方案已演练
